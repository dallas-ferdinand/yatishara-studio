# Ernesto routing

Use **director-ernesto** when the brief needs **character-first conversion**, not witness-object brand memory.

## Route at intake (orchestrator decides once)

| Signal in brief | Director |
|-----------------|----------|
| Brand film, lifestyle, home, comfort, family memory | director-joe |
| Witness object, "that's my life", affinity | director-joe |
| Character must change behavior; relief tomorrow | director-ernesto |
| Conversion, friction removal, "that's happening to me" | director-ernesto |
| Product solves a felt problem through a person | director-ernesto |

When ambiguous, ask Dallas before Phase A.

## Joe vs Ernesto

| | Joe Elliott | Ernesto |
|--|-------------|---------|
| Center | Object witnesses life | Person's behavior changes |
| Question | What object holds this experience? | What behavior proves life got lighter? |
| Buys | Affinity — memory | Conversion — forward relief |
| Time | Backward (memory) | Forward (tomorrow lighter) |

Yatishara default mix: ~30% Joe · ~70% Ernesto. This pipeline picks **one** director per bible.

## director-ernesto enforces

1. **Character arc visible** — observable behavior change across scenes
2. **Forward time** — tomorrow feels lighter, not nostalgic memory
3. **Friction named** — quiet pressure the character moves through
4. **Behavior proof** — same show-don't-tell as Joe; no emotion labels
5. **Conversion beat** — audience sees themselves in the turn, not just the object
